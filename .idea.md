## 1. Title
MWECAU DigiVote: A Secure Online Digital Voting System for Student Elections at Mwenge Catholic University.

## 2. Introduction
The MWECAU ICT Club developed **DigiVote** as its flagship project this semester. Traditional paper-based student elections at MWECAU face issues such as long queues, manual counting errors, low participation, and logistical challenges. DigiVote provides a secure, accessible web platform for multi-level student elections (Presidential, Residential, Course Representative). It was successfully deployed live and is maintained openly on GitHub for transparency and future collaboration.

## 3. Problem Statement
Paper-based elections suffer from:
- Inefficient processes and high administrative burden.
- Risk of vote tampering, miscounts, or loss of ballots.
- Limited accessibility (students must be physically present).
- Low voter turnout due to scheduling conflicts.
- Lack of real-time transparency and result visibility.

These gaps reduce trust and participation in student governance at MWECAU.

## 4. Purpose of the Project
To design, develop, and deploy a secure, user-friendly digital voting platform that replaces paper-based elections, increases participation, ensures vote integrity, and provides real-time results for MWECAU student elections.

## 5. Objectives

**General Objective:**  
Develop and implement a secure web-based voting system for transparent and efficient student elections at MWECAU.

**Specific Objectives:**
- Implement secure voter authentication using one-time tokens and email recovery.
- Support multi-level elections with real-time auto-refresh results.
- Ensure one-voter-one-vote integrity and prevent duplicate voting.
- Deploy a live, accessible system with open-source code on GitHub.
- Document the full development process for club learning and future improvements.

## 6. Justification of the Project
DigiVote contributes to:
- **Efficiency:** Faster voting and instant results.
- **Accessibility:** Students can vote from anywhere with internet access.
- **Transparency & Trust:** Real-time results and public GitHub audit trail.
- **Skill Development:** Club members gain hands-on experience in web development, cybersecurity, and project management.
- **Institutional Impact:** Modernizes student governance at MWECAU and can serve as a model for other clubs or universities in Tanzania.

This project aligns with club member interests in Programming, Cybersecurity, and AI & Machine Learning.

## 7. Literature Review
Traditional paper-based voting and basic Electronic Voting Machines (EVMs) face challenges related to security, scalability, accessibility, and trust. Modern online systems address these through strong authentication (e.g., OTP/one-time tokens), encryption, and real-time processing.

Key insights from existing studies and systems:
- Many university-level e-voting projects use web technologies (PHP/MySQL, Python/Django, or JavaScript stacks) to provide secure authentication and real-time results.
- Common features include voter verification, prevention of duplicate voting, and transparent result dashboards.
- Broader research highlights challenges such as coercion and voter anonymity in large-scale systems, but small-scale controlled campus environments (like ours) can mitigate these risks effectively through registered student accounts.

## 8. Methodology & Approach
- **Approach:** Agile-inspired iterative development with GitHub for version control, issue tracking, and collaboration.
- **Design:** Web-based application with secure backend authentication and a database for voters, candidates, and results.
- **Key Features Implemented:** One-time token authentication, multi-level ballot support, real-time results with auto-refresh, email-based password recovery.
- **Tools/Technologies:** Web server deployment (port 81), Git for version control. All changes tracked publicly.
- **Testing:** Internal club testing + live deployment feedback.
- **Deployment:** Live at http://159.65.119.182:81.

## 9. Scope of the Project

Project Overview

The MWECAU Election Platform is a modern digital voting system designed specifically for Mwenge Catholic University student elections. Built by the MWECAU ICT Club, it transforms traditional paper-based elections into secure, accessible, and efficient digital democracy.

1000+ Students Supported
99.9% System Availability
3 Election Types
6 Residential Areas

🎯 Key Benefits

🌍 Global Access

Students can vote from anywhere in the world - perfect for exchange programs or remote learning situations.

⚡ Lightning Fast

Complete voting process in under 5 minutes with instant result compilation.

🔒 Bank-Level Security

JWT authentication, HTTPS encryption, and anonymous voting ensure complete security.

📱 Mobile First

Responsive design works perfectly on phones, tablets, and computers.

🗳️ Election Types

Presidential Elections: University-wide student leadership for all MWECAU students.

Residential Area Leadership: KIFUMBU, MAWELA, KWACHANGE, WHITE HOUSE, MOSHI MJINI, ON-CAMPUS

Course Leadership: Academic program representatives from undergraduate to doctoral levels.

Quick Start Guide

🚀 For Developers

# Clone the repository git clone https://github.com/mwecauictclub/mwecau_election_platform.git cd mwecau_election_platform # Setup environment cd src cp ../.env.example .env # Initialize database python manage.py migrate # Create admin user python manage.py createsuperuser # Launch development server python manage.py runserver

Step 2: Verification

API Documentation

🔗 Core Endpoints

Authentication

POST /api/auth/login/ # User login POST /api/auth/register/ # User registration POST /api/auth/logout/ # User logout GET /api/auth/user/ # Current user info

Elections

GET /api/elections/ # List all elections GET /api/elections/{id}/ # Election details POST /api/elections/{id}/vote/ # Cast vote GET /api/elections/{id}/results/ # Election results

Commissioner APIs

GET /api/commissioner/stats/ # System statistics GET /api/commissioner/election/{id}/analytics/ # Election analytics POST /api/commissioner/elections/ # Create election PUT /api/commissioner/elections/{id}/ # Update election

📊 Response Formats

Election Object

{ "id": 1, "title": "Presidential Election 2024", "election_type": "presidential", "start_time": "2024-01-15T08:00:00Z", "end_time": "2024-01-15T18:00:00Z", "is_active": true, "total_votes": 245, "candidates": [...] }

Vote Object

{ "election": 1, "candidate": 3, "timestamp": "2024-01-15T10:30:00Z", "voter_hash": "anonymous_hash" }

🔐 Authentication

The API uses JWT (JSON Web Tokens) for authentication. Include the token in the Authorization header:

Authorization: Bearer your-jwt-token-here

📈 Rate Limiting

API endpoints are rate-limited to ensure fair usage and system stability:

  • Authentication: 5 requests per minute
  • Voting: 1 vote per election per user
  • General API: 100 requests per hour

Deployment Guide

🚀 Production Setup

Environment Configuration

# PostgreSQL setup createdb mwecau_elections python manage.py migrate python manage.py createsuperuser

System Architecture

🏗️ Technology Stack

Backend Framework

Django 5.2.7 with custom User model and comprehensive admin interface.

API Framework

Django REST Framework with JWT authentication and permissions.

Database

PostgreSQL for production, SQLite for development.

Frontend

Plain HTML/CSS with responsive design, no framework dependencies.

📊 Database Schema

Core Models: - User (Custom user model) - Election (Presidential, Residential, Course) - Candidate (Election participants) - Vote (Anonymous voting records) - State/Course (Organizational units)

🔄 Application Flow

User Journey

  1. Registration: Student enters registration number
  2. Verification: System validates against student database
  3. Authentication: JWT token issued for session
  4. Election Access: User sees eligible elections
  5. Voting: Anonymous vote cast and recorded
  6. Confirmation: Vote receipt generated
  7. Results: Real-time results after election close

🛡️ Security Architecture

Authentication Layer

JWT tokens with refresh mechanism and secure session management.

Authorization Layer

Role-based permissions with granular access control.

Data Protection

Vote anonymization with audit trails for transparency.

Communication Security

HTTPS/TLS encryption for all data transmission.
